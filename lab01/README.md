Фотостудийная съёмка по шаблонам
Проблема:
Подготовка фотосъёмки в студии требует настройки комплекса взаимосвязанных элементов: схемы освещения, параметров камеры, выбора фона, реквизита, плана съёмки и пресета постобработки. Каждый тип съёмки (Деловой портрет, Семейная, Предметная, Бьюти) диктует свои требования к этим элементам, и все они должны быть согласованы между собой.
При использовании традиционного подхода настройка создаётся с нуля для каждой сессии, что приводит к потере времени и риску ошибок: можно забыть источник света, выбрать неподходящий фон или применить неправильный пресет обработки. Также сложно быстро создавать серию похожих съёмок с небольшими изменениями под конкретных клиентов.
Цель:
Разработать информационную систему с графическим интерфейсом, которая позволяет фотографу/администратору выбрать тип съёмки и автоматически получить согласованный набор всех параметров съёмочной сессии. Система должна поддерживать создание вариаций сессии путём клонирования шаблона, чтобы оригинальный шаблон не изменялся. Также требуется экспорт описания съёмки в HTML-формат.
Описание:
Пользователь выбирает тип съёмки, после чего система формирует объект «СессияСъёмки», содержащий следующие компоненты:
•	СхемаСвета — количество источников света, позиции, мощность, модификаторы (софтбокс, стрипбокс, отражатель)
•	НастройкиКамеры — ISO, выдержка, диафрагма, баланс белого
•	Фон — цвет/текстура/тип (бумажный, тканевый, интерьерный)
•	Реквизит — перечень предметов для сцены (стул, куб, цветы, предметный стол)
•	ПресетОбработки — стиль цветокоррекции и ретуши (neutral, warm, clean, glossy)
•	ЧекЛистПодготовки — список действий перед съёмкой (поставить свет, проверить батареи, подготовить фон)
После создания сессии пользователь может изменять параметры (например, поменять фон или мощность ключевого света) только у копии сессии, не затрагивая исходный шаблон.
Для каждого типа съёмки определён фиксированный набор характеристик компонентов, обеспечивающий целостность сессии:
Тип съёмки	Схема света	Фон	Пресет обработки
Деловой портрет	2–3 источника, контрастный ключевой + заполняющий	Серый/тёмно-синий, нейтральный	Neutral/Contrast, минимальная ретушь
Семейная	Мягкий свет, большой софтбокс, минимум теней	Светлый/тёплый интерьер	Warm/Soft, мягкие тона
Предметная	Направленный свет, контроль бликов, лайткуб при необходимости	Белый/чёрный, чистый	Clean/High-contrast, подчёркивание деталей
Бьюти	Кольцевой/октабокс, акцент на коже, hair/rim light	Однотонный светлый или градиент	Glossy/Beauty, повышенная детализация

Реализация основана на паттерне Прототип (Prototype):
1.	Прототип — интерфейс, определяющий метод clone() для создания копии объекта.
2.	СессияСъёмки — основной класс, реализующий clone() и агрегирующий все компоненты сессии (свет, камера, фон, реквизит, пресет, чек-лист).
3.	СхемаСвета, НастройкиКамеры, Фон, Реквизит, ПресетОбработки, ЧекЛистПодготовки — компоненты, которые также поддерживают clone() для реализации глубокого копирования.
4.	ШаблонДеловогоПортрета, ШаблонСемейнойСъёмки, ШаблонПредметнойСъёмки, ШаблонБьютиСъёмки — конкретные прототипы, представляющие заранее настроенные варианты «СессииСъёмки».
5.	РеестрШаблоновСессий — хранилище прототипов; по запросу возвращает новую сессию через вызов выбранныйШаблон.clone(), после чего пользователь редактирует копию и экспортирует её описание в HTML.
